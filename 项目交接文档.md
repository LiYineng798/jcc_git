# 金铲铲阵容库项目交接文档

## 1. 项目定位

本项目是一个基于 Flask + SQLite + 原生前端（HTML/CSS/JavaScript）的金铲铲阵容码管理 Web 应用。

当前版本已经从“单人本地使用工具”扩展为“多人同时在线使用的轻量级 Web 应用”，支持：

- 游客浏览、搜索、复制阵容码
- 用户注册、登录、验证码校验
- 用户创建、编辑、删除自己的阵容
- 用户点赞、收藏、举报
- 管理员管理用户、管理阵容、处理举报、查看审计日志
- 热度分计算、排行榜等级划分
- 手机端和电脑端自适应
- 白天/夜间模式切换

项目目录：`jcc-lineup-manager`

---

## 2. 技术栈

### 2.1 后端

- Python
- Flask
- SQLite
- Werkzeug（密码哈希等）

### 2.2 前端

- 服务端模板：HTML
- 样式：CSS
- 交互：原生 JavaScript

### 2.3 数据存储

- 默认数据库文件：`instance/lineups.sqlite3`
- 数据库连接配置在 `db.py`
- SQLite 启用了：
  - `PRAGMA foreign_keys = ON`
  - `PRAGMA journal_mode = WAL`
  - `PRAGMA busy_timeout = 5000`

## 3. 当前功能范围

### 3.1 游客能力

- 访问首页查看阵容列表
- 按关键词搜索阵容
- 复制阵容码
- 查看阵容基础热度信息

说明：

- 游客不能新增、编辑、删除阵容
- 游客不能点赞、收藏、举报
- 游客复制会计入热度统计，但按 IP 做 10 分钟去重

### 3.2 普通登录用户能力

- 注册账号
- 登录/退出登录
- 完成人机验证码
- 新增自己的阵容
- 编辑自己的阵容
- 删除自己的阵容
- 点赞阵容
- 收藏阵容
- 举报阵容

说明：

- 用户只能修改/删除自己创建的阵容
- 允许给自己阵容点赞
- 点赞不可取消
- 收藏当前实现只有“收藏”接口，没有“取消收藏”接口

### 3.3 管理员能力

- 访问后台管理页 `/admin`
- 查看用户列表并搜索
- 新建用户
- 修改用户
- 禁用用户
- 查看阵容列表并搜索
- 隐藏/恢复阵容
- 手工调整阵容点赞数和复制数修正值
- 查看待处理举报
- 处理举报并可选隐藏阵容
- 驳回举报
- 查看审计日志

## 5. 后端架构说明

### 5.1 应用入口

`app.py`

职责：

- 创建 Flask 应用
- 加载配置
- 初始化数据库
- 注册各业务蓝图
- 暴露页面路由：
  - `/`
  - `/auth`
  - `/api/health`

### 5.2 认证模块

`auth.py`

职责：

- 注册、登录、退出登录
- 基于 Session 维护当前用户
- 生成和校验 CSRF Token
- 提供 `login_required()`、`admin_required()` 权限判断
- 提供当前登录用户信息接口 `/api/me`

### 5.3 验证码模块

`captcha.py`

职责：

- 读取 `captcha_manifest.json`
- 发放图片验证码挑战
- 校验验证码答案
- 提供测试辅助函数（供自动化测试读取答案）

### 5.4 阵容业务模块

`lineups.py`

职责：

- 阵容列表查询
- 阵容新增/编辑/删除
- 阵容隐藏
- 点赞
- 复制统计
- 收藏
- 举报
- 序列化阵容输出数据

### 5.5 后台管理模块

`admin.py`

职责：

- 用户管理
- 阵容管理
- 举报处理
- 后台统计
- 审计日志查询

### 5.6 数据库模块

`db.py`

职责：

- 管理 SQLite 连接
- 初始化全部表结构
- 引导默认管理员账号
- 自动迁移旧版单人阵容表结构

补充说明：

- 旧版单人应用的 `lineups` 表没有 `user_id` 等字段
- 当前版本会在初始化时自动补齐缺失列
- 历史单人阵容会归属到管理员账号名下

### 5.7 热度计算模块

`scoring.py`

职责：

- 汇总阵容点赞数
- 汇总阵容复制数
- 计算热度分
- 计算阵容等级（SS / S / A / B）

### 5.8 审计与限流

- `audit.py`：记录关键操作日志
- `rate_limit.py`：用于登录/注册等接口的频控

## 9. 关键接口行为说明

### 9.1 `/api/me`

返回：

- 当前登录用户的公开信息
- 当前 Session 的 CSRF Token

前端凡是发起 `POST` / `PUT` / `DELETE`（除登录、注册、验证码校验外），都需要带：

`X-CSRF-Token`

### 9.2 `/api/lineups`

查询行为：

- 支持关键词搜索
- 面向公开列表时，隐藏或删除状态的阵容不会展示给普通访客

新增行为：

- 需要登录
- 阵容会自动记录 `user_id`
- 默认状态为 `normal`

### 9.3 点赞规则

- 允许自己点赞
- 点赞不可取消
- 同一用户对同一阵容每天只能点一次
- 每个用户每天最多点 5 次

### 9.4 复制规则

- 游客和登录用户都可复制
- 登录用户按 `user_id` 去重
- 游客按 IP 去重
- 去重窗口为 10 分钟
- 计分按“是否计数成功”处理

### 9.5 举报规则

- 需要登录
- 举报原因长度限制为 1 到 300 字
- 后台默认只显示待处理举报
- 管理员处理后，举报会从待处理列表中移除

### 9.6 收藏规则

- 当前仅支持收藏
- 当前没有取消收藏接口

## 10. 热度分与等级规则

### 10.1 热度分公式

当前热度分规则为：

```text
score = likes * 5 + counted_copies * 1 + admin_like_adjustment * 5 + admin_copy_adjustment
```

说明：

- `likes`：有效点赞数
- `counted_copies`：有效复制计数
- `admin_like_adjustment`：管理员手工点赞修正
- `admin_copy_adjustment`：管理员手工复制修正

### 10.2 排行统计窗口

- 默认统计最近 7 天的数据

### 10.3 等级划分

基于百分位：

- 前 5%：`SS`
- 接下来 15%：`S`
- 接下来 30%：`A`
- 其余：`B`

## 13. 安全与约束

### 13.1 已实现

- Session 登录
- 密码哈希存储
- CSRF 校验
- 基础接口限流
- 图形验证码
- 用户/管理员权限区分
- 审计日志

### 13.2 生产部署必须注意

- 必须修改默认管理员账号密码
- 必须设置强 `SECRET_KEY`
- 不建议用 Flask 自带开发服务器长期对外提供服务
- 建议使用 `gunicorn` / `uwsgi` + `nginx`

## 16. 当前已知业务规则总结

为了便于接手同事快速理解，这里把业务规则集中列出。

### 16.1 阵容

- 阵容默认公开
- 阵容创建者可编辑、删除自己的阵容
- 管理员可管理所有阵容
- 隐藏状态阵容不会出现在普通用户公开列表

### 16.2 点赞

- 允许自己点赞
- 不可取消点赞
- 每日限额 5 次

### 16.3 复制

- 游客可复制
- 10 分钟窗口去重
- 复制会影响热度分

### 16.4 举报

- 需登录
- 后台默认只展示待处理举报
- 处理举报时可选择同时隐藏阵容

### 16.5 后台搜索

- 用户管理支持按用户名、邮箱、昵称搜索
- 阵容管理支持按阵容名、阵容码、作者用户名、作者昵称搜索
