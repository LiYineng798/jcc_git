# 金铲铲阵容库项目交接文档

## 1. 项目定位

本项目是一个基于 Flask + SQLite + 原生前端（HTML/CSS/JavaScript）的金铲铲阵容码管理 Web 应用。

当前版本已经从“单人本地使用工具”扩展为“多人同时在线使用的轻量级 Web 应用”，支持：

- 游客浏览、搜索、复制阵容码
- 游客查看阵容详情
- 用户注册、登录、验证码校验
- 用户创建、编辑、删除自己的阵容
- 用户点赞、收藏、取消收藏、举报
- 用户查看个人中心、最近浏览、最近复制、我的举报反馈
- 用户访问作者主页，查看作者公开阵容与累计互动
- 管理员管理用户、管理阵容、处理举报、查看审计日志
- 管理员查看增长漏斗与登录转化数据
- 热度分计算、排行榜等级划分
- 上升榜与推荐排序
- 手机端和电脑端自适应
- 白天/夜间模式切换

项目目录：`jcc-lineup-manager`

## 2. 技术栈

### 2.1 后端

- Python
- Flask
- SQLite
- Werkzeug（密码哈希等）

### 2.2 前端

- 服务端模板：HTML
- 样式：CSS
- 交互：原生 JavaScript

### 2.3 数据存储

- 默认数据库文件：`instance/lineups.sqlite3`
- 数据库连接配置在 `db.py`
- SQLite 启用了：
  - `PRAGMA foreign_keys = ON`
  - `PRAGMA journal_mode = WAL`
  - `PRAGMA busy_timeout = 5000`

## 3. 当前功能范围

### 3.1 游客能力

- 访问首页查看阵容列表
- 进入阵容详情页查看单套阵容
- 按关键词搜索阵容
- 复制阵容码
- 查看阵容基础热度信息
- 点击点赞、收藏、举报时触发登录引导
- 本地暂存最近浏览、最近复制，登录后可同步到账号

说明：

- 游客不能新增、编辑、删除阵容
- 游客不能点赞、收藏、举报
- 游客复制会计入热度统计，但按 IP 做 10 分钟去重

### 3.2 普通登录用户能力

- 注册账号
- 登录/退出登录
- 完成人机验证码
- 新增自己的阵容
- 编辑自己的阵容
- 删除自己的阵容
- 点赞阵容
- 收藏阵容
- 取消收藏阵容
- 举报阵容
- 查看个人中心 `/me`
- 查看最近浏览
- 查看最近复制
- 查看我的举报状态与处理时间
- 查看自己的阵容状态（包括被隐藏阵容）

说明：

- 用户只能修改/删除自己创建的阵容
- 允许给自己阵容点赞
- 点赞不可取消
- 收藏可以取消

### 3.3 管理员能力

- 访问后台管理页 `/admin`
- 查看用户列表并搜索
- 新建用户
- 修改用户
- 禁用用户
- 查看阵容列表并搜索
- 隐藏/恢复阵容
- 手工调整阵容点赞数和复制数修正值
- 查看待处理举报
- 处理举报并可选隐藏阵容
- 驳回举报
- 查看审计日志

## 5. 后端架构说明

### 5.1 应用入口

`app.py`

职责：

- 创建 Flask 应用
- 加载配置
- 初始化数据库
- 注册各业务蓝图
- 暴露页面路由：
  - `/`
  - `/auth`
  - `/author/<username>`
  - `/api/health`

### 5.2 认证模块

`auth.py`

职责：

- 注册、登录、退出登录
- 基于 Session 维护当前用户
- 生成和校验 CSRF Token
- 记录注册成功 / 登录成功增长事件
- 提供 `login_required()`、`admin_required()` 权限判断
- 提供当前登录用户信息接口 `/api/me`

### 5.3 验证码模块

`captcha.py`

职责：

- 读取 `captcha_manifest.json`
- 发放图片验证码挑战
- 校验验证码答案
- 提供测试辅助函数（供自动化测试读取答案）

### 5.4 阵容业务模块

`lineups.py`

职责：

- 阵容列表查询
- 作者公开资料接口
- 阵容详情获取
- 阵容新增/编辑/删除
- 阵容隐藏
- 阵容浏览记录
- 点赞
- 复制统计
- 收藏
- 取消收藏
- 举报
- 增长事件接收接口
- 个人中心数据接口
- 序列化阵容输出数据

### 5.5 历史模块

`history.py`

职责：

- 最近浏览记录 upsert
- 最近复制记录 upsert
- 最近浏览/最近复制列表查询
- 游客本地历史同步到账号
### 5.6 后台管理模块

`admin.py`

职责：

- 用户管理
- 阵容管理
- 举报处理
- 后台统计
- 后台增长漏斗统计
- 审计日志查询

### 5.7 数据库模块

`db.py`

职责：

- 管理 SQLite 连接
- 初始化全部表结构
- 引导默认管理员账号
- 自动迁移旧版单人阵容表结构

补充说明：

- 旧版单人应用的 `lineups` 表没有 `user_id` 等字段
- 当前版本会在初始化时自动补齐缺失列
- 历史单人阵容会归属到管理员账号名下

### 5.8 热度计算模块

`scoring.py`

职责：

- 汇总阵容点赞数
- 汇总阵容复制数
- 计算热度分
- 计算阵容等级（SS / S / A / B）
- 计算上升趋势分（rising）

### 5.9 推荐模块

`recommendation.py`

职责：

- 基于热度、新鲜度和“自己阵容降权”生成推荐分
- 为首页 `recommended` 排序提供分值

### 5.10 增长分析模块

`analytics.py`

职责：

- 维护增长事件白名单
- 写入增长埋点事件
- 聚合后台增长漏斗统计

### 5.11 审计与限流

- `audit.py`：记录关键操作日志
- `rate_limit.py`：用于登录/注册等接口的频控

## 9. 关键接口行为说明

### 9.1 `/api/me`

返回：

- 当前登录用户的公开信息
- 当前 Session 的 CSRF Token

前端凡是发起 `POST` / `PUT` / `DELETE`（除登录、注册、验证码校验外），都需要带：

`X-CSRF-Token`

### 9.2 `/api/lineups`

查询行为：

- 支持关键词搜索
- 面向公开列表时，隐藏或删除状态的阵容不会展示给普通访客
- `view=mine` 时，登录用户可以看到自己未删除的阵容，包括被隐藏阵容
- `view=favorites` 时，只返回当前用户收藏的公开阵容
- `sort` 当前支持：
  - `latest`
  - `hot`
  - `rising`
  - `recommended`
  - `ss`

新增行为：

- 需要登录
- 阵容会自动记录 `user_id`
- 默认状态为 `normal`
- 可在上传/编辑时切换为 `hidden`

### 9.3 点赞规则

- 允许自己点赞
- 点赞不可取消
- 同一用户对同一阵容每天只能点一次
- 每个用户每天最多点 5 次

### 9.4 复制规则

- 游客和登录用户都可复制
- 登录用户按 `user_id` 去重
- 游客按 IP 去重
- 去重窗口为 10 分钟
- 计分按“是否计数成功”处理
- 登录用户复制会写入最近复制历史
- 游客复制会先写入本地历史，登录后可同步到账号

### 9.5 举报规则

- 需要登录
- 举报原因长度限制为 1 到 300 字
- 后台默认只显示待处理举报
- 管理员处理后，举报会从待处理列表中移除

### 9.6 收藏规则

- 支持收藏
- 支持取消收藏

### 9.7 最近浏览规则

- 只有进入详情页并触发“查看”动作，才计入最近浏览
- 登录用户直接写库到 `recent_lineup_views`
- 游客先存本地，登录后同步

### 9.8 举报反馈规则

- 举报人可在 `/me` 查看自己提交的举报
- 状态值包括：`pending`、`resolved`、`dismissed`
- 已处理举报会展示 `handled_at` 与阵容当前状态

### 9.9 作者主页规则

- 作者页路由为 `/author/<username>`
- 只展示非管理员用户
- 只返回作者的公开阵容（`status = normal`）
- 不返回邮箱、登录信息、举报信息等敏感字段

### 9.10 增长事件规则

- 前端匿名/登录埋点统一写入 `growth_events`
- 事件名命中白名单才允许写入
- 当前已使用事件包括：
  - `click_login_entry`
  - `open_auth_page`
  - `guest_click_like`
  - `guest_click_favorite`
  - `guest_click_report`
  - `register_success`

## 10. 热度分与等级规则

### 10.1 热度分公式

当前热度分规则为：

```text
score = likes * 5 + counted_copies * 1 + admin_like_adjustment * 5 + admin_copy_adjustment
```

说明：

- `likes`：有效点赞数
- `counted_copies`：有效复制计数
- `admin_like_adjustment`：管理员手工点赞修正
- `admin_copy_adjustment`：管理员手工复制修正

### 10.2 排行统计窗口

- 默认统计最近 7 天的数据

### 10.3 发现层排序

- `hot`：最近 7 天热度最高
- `rising`：最近 24 小时相对前 24 小时增长更快
- `recommended`：基于热度、新鲜度和“自己的阵容降权”的规则式推荐

### 10.4 等级划分

基于百分位：

- 前 5%：`SS`
- 接下来 15%：`S`
- 接下来 30%：`A`
- 其余：`B`

## 16. 当前已知业务规则总结

为了便于接手同事快速理解，这里把业务规则集中列出。

### 16.1 阵容

- 阵容默认公开
- 阵容创建者可编辑、删除自己的阵容
- 管理员可管理所有阵容
- 隐藏状态阵容不会出现在普通用户公开列表

### 16.2 点赞

- 允许自己点赞
- 不可取消点赞
- 每日限额 5 次

### 16.3 复制

- 游客可复制
- 10 分钟窗口去重
- 复制会影响热度分

### 16.4 举报

- 需登录
- 后台默认只展示待处理举报
- 处理举报时可选择同时隐藏阵容

### 16.5 后台搜索

- 用户管理支持按用户名、邮箱、昵称搜索
- 阵容管理支持按阵容名、阵容码、作者用户名、作者昵称搜索
